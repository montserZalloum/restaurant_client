## ١. ملخص تنفيذي

### ١.١ الفكرة بسطر واحد

البرنامج المحلي بيقعد بين برنامج الكاشير والطابعة الحرارية، بيعترض كل فاتورة، بيستخرج رقم الطلب، بيرسل الطلب للسحابة، وبيمرّر الفاتورة للطابعة بدون تأخير.

### ١.٢ الزمن المتوقع للتركيب الكامل

| المرحلة | الزمن (أول ٣ مطاعم) | الزمن (بعد ٥ مطاعم) |
|---|---|---|
| التحضير قبل الذهاب | ٣٠ دقيقة | ١٠ دقايق |
| التركيب الفعلي بالمطعم | ٣-٤ ساعات | ١-٢ ساعة |
| **المجموع** | **~٤ ساعات** | **~٢ ساعة** |

### ١.٣ المخاطر الرئيسية (اقرأها قبل ما تطلع للمطعم)

1. **توقّف الطباعة:** أي خطأ بإعدادات الطابعة أو ترتيب العمليات ممكن يوقف المطعم عن البيع. **لازم خطة طوارئ جاهزة.**
2. **عناوين IP المتعارضة:** لو غيّرت IP بترتيب غلط، رح يكون في تعارض ومشاكل بالشبكة.
3. **برنامج الكاشير غير متوقّع:** بعض البرامج عندها حماية تمنع تثبيت برامج أخرى أو تغيير إعدادات الطابعة.
4. **الإنترنت ضعيف:** بعض المطاعم إنترنتهم متذبذب، الاتصال بالسحابة بيصير غير مستقر.

## ٢. المتطلبات الأساسية (مرة واحدة لكل مشروع)

هدول لازم يكونوا جاهزين **قبل** أي محاولة تركيب بأي مطعم. مش لكل مطعم.

### ٢.١ السحابة منشورة على عنوان عام

#### ليش مهمة؟

البرنامج المحلي ما بيعمل شي مفيد لحاله. بدون سحابة شغّالة:
- ما بتقدر تولّد إعدادات للمطعم
- صفحة الشاشة وصفحة الموظف ما رح تشتغل
- البرنامج ما بقدر يبعت الطلبات لحدا

#### المتطلبات

- السحابة منشورة على سيرفر عام (مش جهاز ببيتك)
- العنوان بشكل: `https://api.queue-manager.com` (مثال)
- شهادة أمان صالحة (HTTPS، مش HTTP عادي)
- متاحة ٢٤/٧

#### كيف تتأكد قبل ما تروح

من جهازك الشخصي (مش من جهاز المطعم):

```powershell
curl https://your-cloud-domain.com/health
```

لازم يرجّع رد ناجح.

### ٢.٢ لوحة الأدمن تعمل

#### الميزات اللي لازم تكون شغّالة

- إنشاء مطعم جديد
- توليد `restaurant.id`
- توليد `api_key`
- توليد `staff_pin`
- تنزيل ملف `config.json` جاهز

#### كيف تتأكد

ادخل على لوحة الأدمن، اضغط "إضافة مطعم" بإسم تجريبي. لازم تنزّل ملف يحتوي:

```json
{
  "restaurant": {
    "id": "rest_xxxxxx",
    "name": "...",
    "api_key": "ak_..."
  },
  "cloud": {
    "base_url": "https://...",
    "ws_url": "wss://...",
    "settings_channel": "settings:rest_xxxxxx"
  },
  "staff_pin": "..."
}
```

### ٢.٣ حزمة تركيب جاهزة

#### كيف تبنيها

من مجلد المشروع على جهازك الشخصي:

```powershell
npm install
npm run vendor
npm run package
```

النتيجة: مجلد `dist\queue-manager-installer\` فيه كل اللي بدّك إياه.

#### تأكد من المحتوى

```powershell
Get-ChildItem dist\queue-manager-installer
```

لازم تشوف:
- `install.bat`
- `uninstall.bat`
- `test-mode.bat`
- `test-printer.bat`
- `setup-helper.js`
- `nssm.exe`
- `node\` (مجلد فيه node.exe)
- `agent\` (فيه `src\` و `node_modules\`)
- `config.json` (نسخة قالب — رح يتم استبدالها لكل مطعم)

## ٣. تحضير حزمة لمطعم محدد

كل مطعم بيلزمه نسخة مستقلة من الحزمة، لأن `config.json` بيكون مختلف.

### ٣.١ ولّد ملف الإعدادات من لوحة الأدمن

1. ادخل على لوحة الأدمن السحابية
2. اضغط "إضافة مطعم جديد"
3. عبّي:
   - اسم المطعم
   - الموقع
   - أي بيانات أخرى تطلبها اللوحة
4. اضغط "حفظ"
5. نزّل ملف `config.json` المولّد

### ٣.٢ ابني حزمة جديدة لهاد المطعم

من جهازك الشخصي:

```powershell
npm run package
```

### ٣.٣ ضع `config.json` بالحزمة

استبدل ملف القالب بالملف اللي نزّلته من اللوحة:

```powershell
Copy-Item -Path "C:\path\to\downloaded\config.json" `
          -Destination "dist\queue-manager-installer\config.json" `
          -Force
```

### ٣.٤ تأكد من صحة الملف

```powershell
node scripts\setup-helper.js verify-config dist\queue-manager-installer\config.json
```

> ⚠️ هاد الفحص رح يفشل بقسم `network` لأنه لسا ما عبيناه بقيم المطعم. هاد طبيعي بهاي المرحلة. باقي الأقسام لازم تكون صحيحة.

### ٣.٥ انسخ الحزمة على USB

```powershell
Copy-Item -Path "dist\queue-manager-installer" `
          -Destination "E:\queue-manager-installer" `
          -Recurse `
          -Force
```

(غيّر `E:\` لحرف الـ USB عندك)

## ٤. قرار معماري: أين سيعمل البرنامج المحلي

البرنامج محتاج جهاز ويندوز يشتغل عليه ٢٤/٧. في ٣ خيارات.

### ٤.١ الخيار أ: جهاز الكاشير نفسه

**المتطلبات:**
- ويندوز ١٠ أو ١١
- صلاحية Administrator
- إذن من صاحب المطعم
- ٢ جيجا RAM وأكثر
- ٥٠٠ ميجا مساحة فاضية

**ميزاته:**
- بدون تكلفة إضافية
- بدون معدّات زيادة بالمطعم

**عيوبه:**
- لو الكاشير وقع، النظام كله بيوقف
- لازم تثبت برنامج جانبي على جهاز ما هو ملكك
- بعض برامج الكاشير عندها حماية تمنع التثبيت

**موصى به:** بأول مطعمين، عشان تختبر النظام بأقل تكلفة.

### ٤.٢ الخيار ب: جهاز مخصّص تجيبه أنت

شراء جهاز صغير مثل:
- Mini PC (Intel NUC، Beelink، Minisforum)
- لاب توب قديم بحالة جيدة

**ميزاته:**
- معزول عن الكاشير
- مضمون من ناحية المواصفات
- بتختبره ببيتك قبل ما تطلع
- لو خرب، بتبدّله بسرعة

**عيوبه:**
- تكلفة ٥٠-١٥٠ دولار لكل جهاز
- معدّة زيادة بالمطعم تحتاج مساحة وكهرباء

**موصى به:** بعد ما تستقر العملية وتبدأ تركّب بكثرة.

### ٤.٣ الخيار ج: Raspberry Pi (مستقبلاً)

**حالياً غير مدعوم.** البرنامج مكتوب لـ ويندوز فقط. الـ Pi رح يحتاج طبقة لينكس مخصّصة (`src/platform/linux/`) وما زالت غير مطوّرة.

**القيمة المستقبلية:**
- جهاز واحد ٥٠-٧٠ دولار يعمل دور:
  - البرنامج المحلي
  - جسر طباعة (لو الطابعة USB)
  - معزول تماماً عن الكاشير
- معماراً أنظف من الخيارين فوق

**التقدير:** ٢-٣ أسابيع تطوير لإضافة دعم لينكس.

### ٤.٤ ملخص القرار

| المرحلة | الخيار الموصى به |
|---|---|
| أول ٣ مطاعم | الخيار أ (جهاز الكاشير) |
| ٤-١٠ مطاعم | الخيار ب (Mini PC) |
| ١٠+ مطاعم | الخيار ج (Pi، يحتاج تطوير) |

## ٥. قرار معماري: نوع الطابعة

النظام بيعمل **بس** مع طابعات شبكية. لو الطابعة USB أو Bluetooth، النظام ما بيشتغل بدون تعديل بالشبكة.

### ٥.١ طابعة شبكية مباشرة (المدعوم ✅)

**العلامات:**
- موصولة بسلك إيثرنت
- لها عنوان IP بالشبكة
- ورقة Self-test بتطلع فيها العنوان والبورت

**هاي الحالة العادية، ما في إجراء إضافي.**

### ٥.٢ طابعة USB (غير مدعوم بدون قطعة إضافية ❌)

**الحلول العملية:**

#### الحل أ: محوّل USB Print Server

اشتري قطعة جاهزة، أمثلة:
- TP-Link TL-PS110U (~٤٠$)
- Silex SX-DS-3000WAN (~٧٠$)
- StarTech PM1115U2 (~٥٠$)

**التركيب:**
1. شبك الطابعة بالـ USB على المحوّل
2. شبك المحوّل بالشبكة بكابل إيثرنت
3. عيّن للمحوّل عنوان IP ثابت
4. عدّل برنامج الكاشير ليطبع على عنوان المحوّل بدل USB

#### الحل ب: Raspberry Pi كجسر طباعة

`Raspberry Pi Zero 2 W` بـ ١٥-٢٠$ + بطاقة SD. تثبت عليه `CUPS` أو سكربت `socat` بسيط، تشبك الطابعة بالـ USB عليه.

أرخص من المحوّل وأمرن، بس بيحتاج إعداد لمرة وحدة بالبيت.

#### الحل ج: استبدال الطابعة بطابعة شبكية

طابعة حرارية شبكية جديدة بـ ١٠٠-٣٠٠$. الحل الأنظف لكن الأغلى.

### ٥.٣ طابعة Bluetooth (غير مدعوم)

نفس مشكلة USB لكن بدون حل بسيط. ينصح باستبدال الطابعة.

## ٧. المعلومات اللي تجمعها داخل المطعم

عبّي هاي المعلومات أول ما توصل، قبل ما تبدأ بأي تعديل.

### ٧.١ معلومات الشبكة (٤ قيم)

من جهاز الكاشير، افتح Command Prompt واكتب:

```
ipconfig
```

دوّر على الكرت الـ Connected (عادة Ethernet). اكتب:

| القيمة | يروح بـ `config.json` تحت |
|---|---|
| `IPv4 Address` | `network.cashier_ip` |
| `Subnet Mask` | (للتحقق فقط، مش بـ config) |
| `Default Gateway` | (للتحقق فقط، مش بـ config) |
| اسم الكرت (مثل `Ethernet`) | `network.interface_name` |

> ⚠️ اسم الكرت لازم يكون مطابق ١٠٠٪. لو في فراغات أو حروف عربية، اكتبهم زي ما هم.

### ٧.٢ معلومات الطابعة

اطبع `Self-test` من الطابعة. الطريقة بتختلف حسب الموديل:
- **عام:** اضغط زر التغذية مع تشغيل الطابعة، وحرّر الزر بعد ثانية
- **Epson:** اضغط زر التغذية لمدة ٣ ثواني
- **Star:** اضغط الزر الأمامي مع تشغيل الطابعة

من ورقة Self-test:

| القيمة | يروح بـ `config.json` تحت |
|---|---|
| عنوان IP الحالي | `network.printer_old_ip` |
| البورت (عادة 9100) | `network.printer_port` |

### ٧.٣ اختار عنوان IP جديد للطابعة

من جهاز الكاشير، جرّب عناوين قريبة:

```
ping 192.168.1.51
ping 192.168.1.52
ping 192.168.1.200
```

أول عنوان بيرجع `Request timed out` → اختاره.

> ⚠️ نصيحة: اختار عنوان عالي (٢٠٠+) خارج نطاق الـ DHCP عشان الراوتر ما يعطيه لجهاز ثاني لاحقاً.

اكتبه: يروح بـ `network.printer_new_ip`.

### ٧.٤ معلومات جهاز الكاشير

| القيمة | كيف تحصل عليها |
|---|---|
| نسخة الويندوز | `Win + Pause` أو `winver` |
| كلمة سر Admin | اطلبها من صاحب المطعم |
| المساحة الفاضية | `Get-Volume` بـ PowerShell |
| الذاكرة | `Win + Pause` يعرض الـ RAM |

**الشروط الدنيا:**
- ويندوز ١٠ أو ١١
- ٢ جيجا RAM
- ٥٠٠ ميجا فاضية

## ٨. الترتيب الصحيح للعمليات داخل المطعم

> 🚨 **القاعدة الذهبية:** ما تعدّل أي إشي إلا لما تكون متأكد إنّو فيك ترجعه ٥ دقايق.

### المرحلة ١: تأكيد الوضع الطبيعي (١٥ دقيقة)

#### الخطوات

1. اطلب من صاحب المطعم يبعت طلب وهمي من الكاشير
2. تأكد إنّو الفاتورة طبعت بنجاح
3. خذ صورة للفاتورة بالموبايل (للمقارنة لاحقاً)
4. اجمع كل المعلومات من القسم ٧

#### فحص النجاح ✅
- [ ] الفاتورة طبعت طبيعي
- [ ] صورة الفاتورة محفوظة
- [ ] نموذج دفتر الملاحظات معبّى كاملاً

#### لو فشل

**الفاتورة ما طبعت أصلاً:**
- المشكلة موجودة قبلك، مش من تركيبك
- ما تكمّل التركيب لحد ما تتحل
- اعرف من صاحب المطعم: هل الكاشير شغّال أم مش شغّال؟

---

### المرحلة ٢: تجهيز ملف الإعدادات (١٠ دقايق)

#### الخطوات

1. شبك الـ USB بجهاز الكاشير
2. انسخ مجلد `queue-manager-installer` لمكان دائم على القرص:
   ```powershell
   Copy-Item -Path "E:\queue-manager-installer" `
             -Destination "C:\queue-manager-installer" `
             -Recurse
   ```
3. افتح `C:\queue-manager-installer\config.json` بـ Notepad
4. عبّي قسم `network` بالقيم اللي جمعتها:
   ```json
   "network": {
     "cashier_ip": "192.168.1.10",
     "printer_old_ip": "192.168.1.50",
     "printer_new_ip": "192.168.1.200",
     "printer_port": 9100,
     "interface_name": "Ethernet"
   }
   ```
5. احفظ الملف

#### فحص النجاح ✅

من Command Prompt كـ Administrator:
```cmd
cd C:\queue-manager-installer
node setup-helper.js verify-config config.json
```

لازم يطلع: `OK` بدون أخطاء.

#### لو فشل

## ٩. خطة الطوارئ (Rollback)

### ٩.١ متى تطبّق

طبّقها لو:
- النظام تركّب بس فيه مشكلة كبيرة ما قدرت تحلها بـ ١٥-٣٠ دقيقة
- الفواتير ما عم تطبع
- المطعم بيستعجل عشان عندهم زبائن
- بدّك ترجع المطعم لوضعه الطبيعي ٢٠ مطلق وتعود لاحقاً

### ٩.٢ خطوات الترجيع (٥-١٠ دقايق)

#### من جهاز الكاشير، Command Prompt كـ Administrator:

1. أوقف الخدمة:
   ```cmd
   net stop QueueManager
   ```

2. أزل خدمة الويندوز:
   ```cmd
   C:\ProgramData\QueueManager\nssm.exe remove QueueManager confirm
   ```

3. أزل الـ IP alias (غيّر القيم لقيمك):
   ```cmd
   netsh interface ip delete address "Ethernet" 192.168.1.50
   ```

4. أزل قواعد الجدار الناري:
   ```cmd
   netsh advfirewall firewall delete rule name="Queue Manager - Local Server"
   ```

#### من إعدادات الطابعة:

5. ارجع الطابعة لعنوانها الأصلي:
   ```
   192.168.1.200  →  192.168.1.50
   ```

6. اطبع Self-test للتأكيد

#### اختبار:

7. اطلب من الكاشير يبعت طلب وهمي. لازم يطبع طبيعي زي قبل ما تجي.

### ٩.٣ ما بعد الترجيع

- اعتذر لصاحب المطعم
- وثّق المشكلة بالتفصيل
- حلّها ببيتك بدون ضغط
- ارجع بموعد جديد

### ٩.٤ ملاحظة مهمة

الترجيع **ما بيحذف** البيانات من السحابة. المطعم لسا موجود بلوحة الأدمن. لما ترجع، إعادة التركيب ما بتحتاج تكرار خطوات لوحة الأدمن.

### الملحق ج: مفاهيم تقنية مفيدة

#### IP Alias

كرت الشبكة الواحد بقدر ياخد أكثر من عنوان IP. هاد بيخلي جهازك يستجيب لعنوانين بنفس الوقت. بنظامنا:
- العنوان الأصلي للجهاز: `192.168.1.10` (مثلاً)
- العنوان البديل (alias): `192.168.1.50` (عنوان الطابعة القديم)

الكاشير لما يبعت لـ `192.168.1.50`، الراوتر بيوصله لجهازك (لأنو هو اللي عنده هاد العنوان)، والبرنامج بيستقبل البيانات على البورت ٩١٠٠.

#### NSSM (Non-Sucking Service Manager)

أداة بتحوّل أي برنامج عادي لخدمة ويندوز. النظام بيستعملها لتحويل برنامج Node.js لخدمة بتشتغل تلقائياً عند تشغيل الويندوز، وبتعيد التشغيل لو وقعت.

#### Encoding (الترميز)

الطابعات الحرارية القديمة بتستعمل ترميزات مختلفة لتمثيل الحروف العربية:
- **UTF-8:** الترميز الحديث، يدعم كل اللغات
- **Windows-1256:** ترميز قديم لويندوز العربي
- **CP864:** ترميز قديم جداً لـ DOS العربي

البرنامج بيحاول يكتشف الترميز تلقائياً، بس لو فشل، الفنّي بيحدّده يدوياً بـ `config.json`.

#### Service Recovery

ويندوز بيقدر يعيد تشغيل الخدمات لما تفشل. النظام معدّ عشان يعيد التشغيل:
- بعد ٥ ثواني للفشل الأول
- بعد ١٠ ثواني للفشل الثاني
- بعد ٢٠ ثانية للفشل الثالث
- بحد أقصى ١٠ مرات بـ ٣٠ دقيقة
